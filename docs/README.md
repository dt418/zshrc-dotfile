# Dotfiles Documentation

Welcome to the dotfiles documentation! This guide is organized using the [Diátaxis framework](https://diataxis.fr/) to help you find the right information for your needs.

## Documentation Structure

### [Tutorials](./tutorials/)
Learning-oriented guides for getting started with the dotfiles system.
- **Getting started**: For new users setting up the dotfiles
- **Installation guide**: Step-by-step installation process

### [How-to guides](./how-to/)
Task-oriented guides for achieving specific goals.
- **Add new aliases**: How to add and test new shell aliases
- **Add new functions**: How to add and test new shell functions
- **Run tests**: How to validate your dotfiles configuration
- **Update dotfiles**: How to keep your dotfiles up to date

### [Reference](./reference/)
Information-oriented reference materials.
- **Commands**: Complete list of available make commands and their usage
- **Architecture**: Detailed explanation of the dotfiles structure and module loading order
- **Code style**: Coding conventions and guidelines for Zsh scripts
- **Commit conventions**: Guide to writing proper commit messages

### [Explanation](./explanation/)
Understanding-oriented discussions.
- **Design principles**: Why the dotfiles are structured this way
- **Module system**: How the modular Zsh configuration works
